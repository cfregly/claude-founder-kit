# Receipts

Receipts are evidence files. They do not all carry the same weight.

## Receipt Labels

- `internal simulation`: the repo was run against a realistic scenario by the maintainer or agent.
- `mechanical receipt`: a deterministic command produced a score, report, or cost table.
- `external value receipt`: a skeptical user tested a stage against a baseline workflow and recorded the outcome.
- `rejected`: the run did not help or made the workflow worse.

Only an external value receipt can support a claim that a stage was adversarially-confirmed to add value.
