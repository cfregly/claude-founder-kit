# Internal Founder Simulation Scenario

Status: internal adversarial simulation, not external proof.

Reviewer role: skeptical internal founder-operator.

Company: LedgerOps AI.

Workflow: a seed-stage founder is building a Claude support copilot for finance teams. The product answers questions over ERP tickets, policy docs, and billing events, then proposes safe next actions for support and success teams.

Baseline process:

1. Founder writes a short pitch and picks the first product wedge from instinct.
2. Founder creates a small happy-path eval set after the demo works.
3. Founder drafts MCP tools quickly so the agent can search tickets, update cases, query billing, and issue refunds.
4. Founder plans launch from a sample cohort spreadsheet.

Adversarial questions:

- Does the pitch name a narrow enough first buyer?
- Does the eval set catch hallucination and instruction override, or only happy paths?
- Does the tool surface expose raw secrets, destructive actions, or raw query escape hatches?
- Does the launch and scale readout produce a concrete next motion instead of generic GTM advice?
